[[security-api-disable-user-profile]]
=== Disable user profile API

beta::[]

++++
<titleabbrev>Disable user profile</titleabbrev>
++++

Disable th user profile that's associated with the specified unique ID.
The user profile is **not** visible to search once disabled.

[[security-api-disable-user-profile-request]]
==== {api-request-title}

`POST /_security/profile/<uid>/_disable`

[[security-api-disable-user-profile-prereqs]]
==== {api-prereq-title}

To use this API, you must have at least the `manage_user_profile` cluster privilege.

[[security-api-disable-user-profile-desc]]
==== {api-description-title}

When activating a user profile with <<security-api-activiate-user-profile>>,
the profile becomes enabled. You can use this Disable User Profile API and
the <<security-api-enable-user-profile>> to change the `enabled` status.

[[security-api-disable-user-profile-path-params]]
==== {api-path-parms-title}

`uid`::
(Required, string) A unique identifier for the user profile.
