[[indices-close]]
== Close index API
++++
<titleabbrev>Close index</titleabbrev>
++++

Closes an open index.

[float]
[[indices-close-request]]
=== {api-request-title}

`POST /<index>/_close`

[float]
[[indices-close-desc]]
=== {api-description-title}

You can use the close index API to close an <<indices-open, open index>>.

{es} blocks read and write operations for closed indices. You cannot index or
search for documents in closed indices. You can re-open a closed index using the
<<indices-open,open index API>>.

Because closed indices do not maintain internal data structures for indexing or
searching documents, they have a smaller overhead on the cluster.

When closing an index, the <<master-node,master node>> restarts the index shards
to reflect the new state of the index. The index shards then go through the
<<indices-recovery,index recovery process>>. The cluster automatically copies
the data of closed indices to ensure a safe number of shard copies at all times.

NOTE: Closed indices can consume a significant amount of disk-space, which may
cause problems in managed environments. You can disable index closing by
changing the <<cluster-indices-close,`cluster.indices.close.enable` setting>>
to `false`.

[float]
[[indices-close-path-params]]
=== {api-path-parms-title}

`<index>` (Required)::
+
--
(string) Comma-separated list or wildcard expression of index names to retrieve.

If the `action.destructive_requires_name` setting is `true`, you can use a value
of `_all` or `*` to close all indices.
--

[float]
[[indices-close-query-params]]
=== {api-query-parms-title}

`allow_no_indices` (Optional)::
(boolean) Indicates whether the operation is skipped if a wildcard expression
matches no indices. Defaults to `false`.

`expand_wildcards` (Optional)::
(string) Indicates whether wildcard expressions should expand to
open or closed indices. Possible values are:

* `open` (Default)
* `closed`
* `none`
* `all`

`ignore_unavailable` (Optional)::
(boolean) Indicates whether unavailable indices are skipped. Defaults to
`false`.

`wait_for_active_shards` (Optional)::
(integer) Number of active shard copies required to start before returning a
response. See <<index-wait-for-active-shards>>.
