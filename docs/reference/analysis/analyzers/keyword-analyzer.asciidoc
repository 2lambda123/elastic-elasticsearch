=== Keyword Analyzer

The `keyword` analyzer simply treats the entire text stream as a
single token. This is useful for data like zip codes, IDs, and so on.
Note that you can also simply configure a field as `not_analyzed` in 
your mappings.

The `keyword` analyzer has no configurable attributes.

For example, using the `keyword` analyzer to process the string 
"bit.ly/1wsCiKv" generates one token:

`bit.ly/1wsCiKv`
