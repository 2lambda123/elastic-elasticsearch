[[analysis-snowball-analyzer]]
=== Snowball Analyzer

The `snowball` analyzer wraps the <<analysis-standard-tokenizer,`standard`>> 
tokenizer and the <<analysis-standard-tokenfilter,`standard`>>,
<<analysis-lowercase-tokenfilter,`lowercase`>>,
<<analysis-stop-tokenfilter,`stop`>>, and
<<analysis-snowball-tokenfilter,`snowball`>> token filters. The `standard` 
tokenizer splits tokens on whitespace and drops most punctuation. Periods 
that are not followed by whitespace, such as in a URL, are preserved. All 
tokens are converted to lowercase by the `lowercase` token filter, and any 
specified stop words are removed by the `stop` token filter. The `snowball` 
token filter then stems the tokens using a language specific 
https://github.com/snowballstem[Snowball] generated stemmer. The `snowball` 
analyzer does not use any character filters.

You can configure the following settings for a `snowball` analyzer:

[cols="<,<",options="header",]
|=======================================================================
|Setting |Description
|`language` |The language of the content being stemmed. Defaults to `English`.
See the <<analysis-snowball-tokenfilter, Snowball Token Filter>> for the list
of supported languages.
|`stopwords` |A list of stopwords. Default stopwords are defined for many,
but not all languages. You can use this attribute to specify stopwords for
languages that have no defaults, or replace the default stopword list. See
the https://github.com/apache/lucene-solr/tree/trunk/lucene/analysis/common/src/resources/org/apache/lucene/analysis/[Apache Lucene-Solr analysis 
resources] for the stopword defaults for each language.
|=======================================================================

For example, using the `snowball` analyzer to process the string 
"There is no way that writers can be tamed and rendered civilized" with the 
default stopwords generates the following tokens:

`way, writer, can, tame, render, civil`
